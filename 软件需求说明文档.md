# 软件功能需求文档 (FRD)：通用层次贝叶斯网络效能评估系统

## 1. 产品定位与目标
[cite_start]本系统旨在通过构建多层贝叶斯概率图模型，评估具有嵌套层级关系（如：政策/战略层 -> 社会/战役层 -> 行为/战术层）的组织效能 [cite: 3, 230][cite_start]。系统通过数学算法剥离环境变量的影响，精准定位实际效能偏离期望值的“亮点 (Bright Spots)”与“暗点 (Dark Spots)” [cite: 5]，广泛适用于公共安全、社会治理等领域的复杂数据推断。适配多平台

## 2. 系统架构 (Tech Stack)
* **前端展示层**: Vue3 (Composition API) + Tailwind CSS + Apache ECharts。
* **宿主/通信层**: Tauri (Rust)，提供跨平台原生窗口及系统级轻量化支持。
* **后端算法层**: FastAPI 提供 HTTP 接口，核心计算由 PyMC (v5+) 承担。
* **硬件加速**: 算法引擎需配置 JAX 后端，以充分榨取 Precision 7960 Tower 级别工作站及双路 NVIDIA RTX A6000 的并行张量计算算力。也要兼容普通平台

## 3. 核心功能模块 (Core Modules)

### 模块 A：双轨制智能数据接入中心 (Frontend)
* **数据导入**: 支持标准 CSV/Excel 宽表的一键上传，并提供通用模板下载功能。也可以支持手动录入（要用户友好）
* **启发式盲猜 (Heuristic Guessing)**: 读取表头后，利用正则表达式自动识别层级 ID（如“市局”、“分局”）、协变量 $X$（如“资金”、“得分”）以及观测指标 $Y$。
* **拓扑可视化拖拽**: 在界面生成“层级树”，支持用户通过双栏拖拽的方式，修正变量映射并支持无限层级的动态增删。

### 模块 B：数据健康体检安全阀 (Frontend)
* **拦截与质检**: 在组装 JSON 提交给后端前执行本地扫描。
* **致命错误拦截**: 检查结构断层（缺失 ID 列）、严重类型错误（非数值数据混入），若存在则阻断运行。
* **预警与自动修复**: 扫描缺失值与“零方差”变量，提供“一键均值填充”或“一键移除该列”的快捷交互。

### 模块 C：动态贝叶斯建图工厂 (Backend)
* **接口网关**: FastAPI 接收前端动态 JSON，利用 Pandas 进行分类变量的整数编码 (`Categorical.codes`) 与特征矩阵标准化 (Z-score)。
* **非中心化重构 (Non-centered Parameterization)**: 利用 Numpy 张量加法广播 (`Additive Broadcast`) 取代递归树，动态构建多层截距，彻底消除多层级网络采样中的“尼尔漏斗”问题。
* [cite_start]**厚尾分布似然函数**: 观测层强制使用自由度为 3.5 的 Student-T 分布 (`nu=3.5`) 处理实际数据 [cite: 236]，以捕获真实世界中的极端超常或失常发挥。

### 模块 D：业务洞察与可视化输出 (Full Stack)
* [cite_start]**极值提取算法**: 从 MCMC `trace` 中提取底层期望均值 $\mu$。将实际值与 $\mu$ 比较，偏差绝对值超过 2 个标准差（$Deviation > 2 \times SD$）的节点标记为亮点或暗点 [cite: 5, 57]。
* [cite_start]**偏差散点图 (Outlier Plot)**: ECharts 渲染。X 轴为模型期望值，Y 轴为偏差量。亮点使用高亮黄色大圆点，暗点使用黑色大圆点 [cite: 44]，符合预期的中庸节点使用灰色空心圆圈。
* **森林图/毛毛虫图 (Caterpillar Plot)**: ECharts 渲染。展示各协变量的影响权重（ $\beta$ ）及其 95% 最高密度区间 (HDI)，帮助用户进行驱动力归因分析。
* **一键导出**: 提供将识别出的异常单位名单及 ECharts 图表导出为 PDF 或高清图片的功能，无缝对接汇报与教学场景。

- 额外的可视化：

一、 贝叶斯结构图（DAG）绝对需要可视化！在动态建图中，我们根据用户的 JSON 自动生成了未知的多层网络。用户肯定会好奇：“系统到底理解对了吗？”在后端 Python (PyMC) 中，我们完全可以一键生成这个结构图（即有向无环图 DAG，Directed Acyclic Graph）。实现方案：PyMC 原生支持 pm.model_to_graphviz(model) 函数。当我们的 build_dynamic_pymc_model 运行完毕后，在开始 MCMC 采样前，后端可以直接调用这个函数，将网络结构渲染成一张 SVG 矢量图，或者将其转化为前端（如使用 X6 或 GoJS 库）可以读取的 JSON 节点连线数据。用户体验 (UX) 价值：结构确认：在界面的“模型状态”面板，渲染出这张带有“盘子（Plates，表示批量数据的层级嵌套，如 $N_{派出所}$）”和节点分布形态（如 Student-T、Normal）的学术级概率图。降维打击的专业感：对于参加“教学创新大赛”或向上级汇报而言，一张随着用户拖拽自动生成的贝叶斯概率图谱，具有极强的极客美学和学术说服力。它明明白白地告诉受众：这不是简单的加权平均，而是真正的底层因果与概率网络。

二、 结果可视化的“核武器”扩展除了我们之前聊过的偏差散点图（找亮点/暗点）和毛毛虫森林图（找驱动力），要让这款软件达到商业级（甚至顶级 BI 工具）的水平，我们还需要在前端 ECharts 面板中补充以下几个维度的增强可视化：
1. 后验预测检查图 (Posterior Predictive Checks, PPC) —— 证明“模型是靠谱的”在《Nature》的那篇论文中，作者专门用后验预测检查和贝叶斯 P 值（BpV）来评估模型的拟合优度 。业务痛点：用户会问，“你算出来的这些亮点，不会是模型瞎猜的吧？”可视化方案：画一张 KDE（核密度估计）曲线图。深蓝色的实线代表真实的破案转化率分布，而后面叠加上百条浅蓝色的半透明虚线，代表模型模拟出的破案转化率分布。解说语境：“大家看，模型推演出的模拟分布网（浅色）完美地包裹住了现实的真实分布（深色）。这在统计学上证明了，我们的评估体系不是空中楼阁，它是极其贴合实战客观规律的。”
2. GIS 空间地理映射 (Geospatial Heatmap) —— 从“散点”到“地理区”。论文中也将全球的珊瑚礁“亮点”和“暗点”投射到了世界地图上 。可视化方案：在 ECharts 中引入 GeoJSON（如某市的行政区划地图）。联动交互：当用户在散点图上点击某个标黑的“暗点”时，右侧的 GIS 地图上对应的辖区（立刻泛起红色的涟漪预警。业务价值：决策者可以直观地看到“暗点”是否呈现空间聚集性（比如城乡结合部的所普遍偏低），从而制定区域性的战役倾斜策略。
3. 互动式“反事实推演”模拟器 (What-If Simulator) —— 把“死报告”变“活沙盘”这是数据分析软件的终极形态，也是极其契合你应用场景的杀手锏。既然我们算出了各个协变量的权重 $\beta$，我们就可以做沙盘推演。可视化方案：在面板下方提供一排滑动条（Sliders），对应前端输入的那些协变量（如：图侦资金投入、单兵考核平均分）。联动交互：用户拖动“单兵考核平均分”的滑块，将其从全局平均的 75 分拉高到 90 分。此时，ECharts 里的散点图开始实时流动——系统根据贝叶斯后验参数，动态重算每一家派出所的期望转化率。

## 4. 性能与合规要求
* **UI/UX**: 遵循“渐进式明细”原则，隐藏底层数学超参数，确保非算法专业用户通过引导即可完成分析。同时对部分功能或者参数要给出必要的提示说明，
* **系统兼容性**: 打包产物需完美支持各类桌面环境（可以使用 GitHub自动打包发布）。
* **异常处理**: 贯彻 Fail-Fast 原则，后端捕获到任何张量维度不匹配或 `LinAlgError`，必须通过 FastAPI 抛出通俗易懂的中文报错至前端。
