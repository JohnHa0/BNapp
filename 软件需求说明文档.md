### 一、 项目代码文件总结与目录结构

目前我们在逻辑上已经完成了前后端核心骨架的探讨，整套系统的物理形态是一个 **Tauri Sidecar 架构**（前端用 Web 技术栈，后端挂载 Python 独立进程）。

建议在 Antigravity 中初始化的项目结构如下：

```text
hb-eval-system/                     # 项目根目录
├── package.json                    # 前端依赖 (Vue3, Tailwind, ECharts)
├── tailwind.config.js              # Tailwind 样式配置
├── src/                            # 前端源码区 (Vue3 + Vite)
│   ├── App.vue                     # 全局入口
│   ├── main.js                     # 挂载路由与插件
│   └── components/
│       ├── DataImporter.vue        # [已提供] 数据接入与智能映射面板
│       ├── DataHealthCheck.vue     # [已提供] 数据体检红绿灯面板
│       └── VisualizationDashboard.vue # [待生成] 偏差散点图与森林图渲染面板
├── src-tauri/                      # Tauri 核心配置区
│   ├── tauri.conf.json             # 核心配置文件 (需在此配置 Python sidecar)
│   ├── Cargo.toml                  # Rust 依赖
│   └── src/main.rs                 # Rust 进程入口 (负责拉起 Python 后端)
└── backend/                        # 算法引擎源码区 (Python)
    ├── requirements.txt            # Python 依赖 (pymc, fastapi, pandas, jax)
    ├── main.py                     # [已提供] FastAPI 网关与数据解析
    ├── model_factory.py            # [已提供] PyMC 动态非中心化建图工厂
    └── analyzer.py                 # [已提供核心逻辑] 提取 trace 结果，计算亮点/暗点
```

#### 初始化核心配置建议

1.  **Python 环境 (`backend/requirements.txt`)**:
    ```text
    fastapi==0.104.1
    uvicorn==0.24.0
    pydantic==2.5.2
    pandas==2.1.3
    numpy==1.26.2
    pymc>=5.10.0
    arviz==0.16.1
    numpyro==0.13.0  # 必须包含，用于激活 GPU 加速
    jax[cuda12]      # 必须包含，适配 NVIDIA RTX 架构
    ```
2.  **Tauri Sidecar 挂载 (`src-tauri/tauri.conf.json` 片段)**:
    在打包阶段，需要利用 PyInstaller 将 `backend/main.py` 打包成单文件可执行程序，并在 Tauri 中声明为外部二进制资源（sidecar），以确保在 Ubuntu 或银河麒麟等 Linux 操作系统上可以“开箱即用”。

### 二、 功能需求文档 (FRD)

# 软件功能需求文档 (FRD)：通用层次贝叶斯网络效能评估系统

## 1. 产品定位与目标
[cite_start]本系统旨在通过构建多层贝叶斯概率图模型，评估具有嵌套层级关系（如：政策/战略层 -> 社会/战役层 -> 行为/战术层）的组织效能 [cite: 3, 230][cite_start]。系统通过数学算法剥离环境变量的影响，精准定位实际效能偏离期望值的“亮点 (Bright Spots)”与“暗点 (Dark Spots)” [cite: 5]，广泛适用于公共安全（如图侦实战效能评估）、社会治理等领域的复杂数据推断。适配windows7以上系统

## 2. 系统架构 (Tech Stack)
* **前端展示层**: Vue3 (Composition API) + Tailwind CSS + Apache ECharts。
* **宿主/通信层**: Tauri (Rust)，提供跨平台原生窗口及系统级轻量化支持。
* **后端算法层**: FastAPI 提供 HTTP 接口，核心计算由 PyMC (v5+) 承担。
* **硬件加速**: 算法引擎需配置 JAX 后端，以充分榨取 Precision 7960 Tower 级别工作站及双路 NVIDIA RTX A6000 的并行张量计算算力。

## 3. 核心功能模块 (Core Modules)

### 模块 A：双轨制智能数据接入中心 (Frontend)
* **数据导入**: 支持标准 CSV/Excel 宽表的一键上传，并提供通用模板下载功能。
* **启发式盲猜 (Heuristic Guessing)**: 读取表头后，利用正则表达式自动识别层级 ID（如“市局”、“分局”）、协变量 $X$（如“资金”、“得分”）以及观测指标 $Y$。
* **拓扑可视化拖拽**: 在界面生成“层级树”，支持用户通过双栏拖拽的方式，修正变量映射并支持无限层级的动态增删。

### 模块 B：数据健康体检安全阀 (Frontend)
* **拦截与质检**: 在组装 JSON 提交给后端前执行本地扫描。
* **致命错误拦截**: 检查结构断层（缺失 ID 列）、严重类型错误（非数值数据混入），若存在则阻断运行。
* **预警与自动修复**: 扫描缺失值与“零方差”变量，提供“一键均值填充”或“一键移除该列”的快捷交互。

### 模块 C：动态贝叶斯建图工厂 (Backend)
* **接口网关**: FastAPI 接收前端动态 JSON，利用 Pandas 进行分类变量的整数编码 (`Categorical.codes`) 与特征矩阵标准化 (Z-score)。
* **非中心化重构 (Non-centered Parameterization)**: 利用 Numpy 张量加法广播 (`Additive Broadcast`) 取代递归树，动态构建多层截距，彻底消除多层级网络采样中的“尼尔漏斗”问题。
* [cite_start]**厚尾分布似然函数**: 观测层强制使用自由度为 3.5 的 Student-T 分布 (`nu=3.5`) 处理实际数据 [cite: 236]，以捕获真实世界中的极端超常或失常发挥。

### 模块 D：业务洞察与可视化输出 (Full Stack)
* [cite_start]**极值提取算法**: 从 MCMC `trace` 中提取底层期望均值 $\mu$。将实际值与 $\mu$ 比较，偏差绝对值超过 2 个标准差（$Deviation > 2 \times SD$）的节点标记为亮点或暗点 [cite: 5, 57]。
* [cite_start]**偏差散点图 (Outlier Plot)**: ECharts 渲染。X 轴为模型期望值，Y 轴为偏差量。亮点使用高亮黄色大圆点，暗点使用黑色大圆点 [cite: 44]，符合预期的中庸节点使用灰色空心圆圈。
* **森林图/毛毛虫图 (Caterpillar Plot)**: ECharts 渲染。展示各协变量的影响权重（ $\beta$ ）及其 95% 最高密度区间 (HDI)，帮助用户进行驱动力归因分析。
* **一键导出**: 提供将识别出的异常单位名单及 ECharts 图表导出为 PDF 或高清图片的功能，无缝对接汇报与教学场景。

## 4. 性能与合规要求
* **UI/UX**: 遵循“渐进式明细”原则，隐藏底层数学超参数，确保非算法专业用户通过引导即可完成分析。
* **系统兼容性**: 打包产物需完美支持 Linux 桌面环境（Ubuntu/银河麒麟）。
* **异常处理**: 贯彻 Fail-Fast 原则，后端捕获到任何张量维度不匹配或 `LinAlgError`，必须通过 FastAPI 抛出通俗易懂的中文报错至前端。
